# GASH Streetwear & Fashion Storefront

**GASH** is a premium, state-of-the-art e-commerce storefront designed for modern streetwear and fashion enthusiasts. Crafted with vibrant layouts, fluid micro-animations, and interactive shopping experiences, GASH redefines online retail. It bridges the gap between digital shopping and community engagement with integrated real-time livestream shopping, lightning-fast passwordless security, and personalized discovery feeds.

---

## 🚀 Key Features

- **Authentication & Security**:
  - OAuth integration (Google Login).
  - Passwordless Passkey Authentication (`@simplewebauthn/browser`).
  - OTP verification email flows powered by **EmailJS**.
  - Modular Authentication Layout using the reusable `AuthTemplate` component.
- **Real-time Collaboration & Streaming**:
  - Livestream capabilities powered by **LiveKit** integration.
  - Real-time event communications via **Socket.IO**.
- **State Management & UI**:
  - State management utilizing **Redux** (`react-redux`).
  - Polished styling with **Tailwind CSS v4** & custom CSS theme tokens.
  - Highly interactive micro-interactions powered by **Framer Motion**.
  - Rich components built with **Material-UI (MUI)**.
- **Utility & Export Tools**:
  - Exporting metrics/tables to Excel (`xlsx`) or PDF (`jspdf`, `html2canvas`).
  - File saving capabilities.
  - Dynamic Date and Time utilities (`date-fns`).

## 🛠️ Tech Stack

- **Framework**: [React 19](https://react.dev/)
- **Build Tool**: [Vite 7](https://vite.dev/)

- **Routing**: [React Router v7](https://reactrouter.com/)
- **Icons**: [Lucide React](https://lucide.dev/) & [Iconify](https://iconify.design/)

## 📂 Directory Structure

```text
src/
├── assets/          # Static assets (images, logos, icons)
├── common/          # Global styles, shared types/components
├── components/      # Reusable UI elements (Button, Form, Header, Layout)
├── config/          # Configurations (API routing, SDK initializations)
├── constants/       # Global constants, messages, configuration values
├── context/         # React Context providers (AuthContext, SocketContext)
├── features/        # Feature-specific hooks, selectors, components (e.g. auth hooks)
├── hooks/           # Custom React hooks (e.g. useToast)
├── mock/            # Mock datasets & configurations for local testing
├── pages/           # Page containers
│   ├── auth/        # Authentication pages (Login, Signup, Register, OTPVerification) utilizing AuthTemplate
│   └── customer/    # Customer-facing views (Cart, Checkout, Profile, Favorite)
├── styles/          # Traditional CSS stylesheets (Login.css, etc.)
└── utils/           # Helper scripts and utility functions (date helpers, exports)
```

## 💻 Running the Application

### 📦 Prerequisites

- **Node.js**: `v18.x` or later (Recommended `v20.x+`)
- **Package Manager**: `npm`

### ⚡ Installation

```bash
npm install
```

### 🏃‍♂️ Local Development Server

Starts the Vite dev server with Hot Module Replacement (HMR).

```bash
npm run dev
```

### 🏗️ Production Build

Compiles and optimizes the app for deployment.

```bash
npm run build
```

### 🔍 Preview Production Build

Locally preview the output generated by the build process.

```bash
npm run preview
```

### 🚨 Linter

Run ESLint to analyze codebase for issues.

```bash
npm run lint
```
